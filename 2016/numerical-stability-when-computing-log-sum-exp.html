Numerical stability when computing log-sum-exp

Mon 07 November 2016

This little trick has saved me a few times. A couple of other blogs also have posted this trick (like this one) but I thought I might as well post it here anyway.

When writing code for probabilistic graphical models it is quite common that you will have to compute a value with the form [\log \sum_i \exp{\phi_i}] where $\phi$ is some kind of potential function. These exponents can be large and when summing over a large number of them it is likely that you will experience numerical overflow before being able to compute the log. Fortunately there is a trick to prevent this from happening. Simple use the identity $$\log \sum_i \exp{\phi_i} = A + \log \sum_i \exp{\phi_i - A}$$ Here $A$ can be anything you like but a good choice is $A=\underset{i}{\max}(\phi_i)$. This should sufficiently shrink the exponent and allow you to compute it without difficulties.
